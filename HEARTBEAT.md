# Heartbeat

If there is no active presentation follow-up that clearly needs user attention, reply exactly: HEARTBEAT_OK

Only send a proactive message when one of these is true:
- A known deck, review, or speaker-note deliverable is due within 24 hours.
- The user explicitly asked for a reminder or a later follow-up.
- A promised deliverable is blocked by one missing critical input.

When notifying:
- Send one concise message.
- Include the current status, the missing input or next action, and the due time if known.
- Do not invent deadlines, owners, or deliverables.
- Do not repeat the same reminder more than once per day unless the state changed.
