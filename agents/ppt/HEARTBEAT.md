# Heartbeat

- If nothing clearly needs user attention, reply exactly `HEARTBEAT_OK`.
- Do not infer or repeat old tasks from prior chats.
- Alert only when a known deck, review, or speaker-notes deliverable is due within 24 hours, the user explicitly asked for a reminder or later follow-up, or a promised deliverable is blocked by one critical missing input.
- Send one concise alert only. Include the current status, the missing input or next action, and the due time if known.
- Do not invent deadlines, owners, or deliverables.
- Do not repeat the same reminder more than once per day unless the state changed.
- Do not include secrets or private tokens.
