# PPT Review

Use this skill when the user already has deck content and needs critique, scoring, or improvement guidance.

## Review Objectives

Evaluate the material for clarity, persuasion, density, evidence quality, and decision-readiness.

## Review Rubric

Assess the following dimensions:

- audience fit
- storyline coherence
- headline strength
- evidence sufficiency
- slide economy
- recommendation clarity
- delivery readiness

## Progress Reporting

For longer review tasks, report progress as the review advances.

Recommended milestones:

- review context understood
- top-level diagnostic complete
- slide-level issues consolidated
- rewrite recommendations drafted
- final verdict packaged

Each progress update should include:

- completed review stage
- active review focus
- next review action
- blocker or missing context, if any

## Review Workflow

### 1. Identify context

Capture:

- audience
- meeting type
- desired outcome
- time limit
- current draft maturity

### 2. Diagnose slide-level issues

Look for:

- titles that describe topics instead of messages
- unsupported claims
- walls of text
- repeated points
- buried recommendations
- abrupt transitions
- weak final ask

### 3. Return prioritized feedback

Structure findings as:

- critical fixes
- important improvements
- optional polish

### 4. Rewrite selectively

When helpful, rewrite:

- slide titles
- opening summary
- recommendation slide
- closing ask

## Output Standard

Default output order:

1. verdict summary
2. scored rubric
3. highest-priority issues
4. suggested rewrites
5. final review checklist
